Описание проекта
SteamMatch — это веб-сервис, который анализирует два открытых Steam-профиля и определяет, насколько пользователи похожи как игроки.
Пользователь вводит свой SteamID (или ссылку на профиль), затем выбирает друга из списка Steam или вставляет его профиль. После анализа сервис строит подробный отчет и рассчитывает общий показатель SteamMatch Score.
Главная идея проекта — не просто сравнить библиотеки игр, а показать, насколько пользователи совместимы как игровые партнеры, какие у них общие интересы и во что им стоит играть вместе.
 
MVP
Минимальная версия продукта должна включать:
ввод SteamID;
выбор друга / ввод еще одного SteamID;
анализ библиотек;
SteamMatch Score;
сравнение игровых часов;
сравнение жанров;
общие игры;
рекомендации для совместной игры;
AI Summary.

 
Основные источники данных
Steam Web API
Используется для получения:
списка игр пользователя;
общего времени в каждой игре (playtime_forever);
времени за последние 2 недели (playtime_2weeks);
списка друзей;
информации о профиле;
достижений (если профиль открыт).

Steam Store API
Позволяет получать информацию о каждой игре:
жанры;
категории;
разработчика;
издателя;
дату выхода;
описание;
изображения.

Дополнительные открытые сервисы
При желании можно использовать:
SteamDB — дополнительная информация об играх и исторических ценах.
SteamSpy — примерные оценки популярности, жанров и аудитории игр.
IsThereAnyDeal — текущие и исторические цены.
Эти сервисы позволяют сделать аналитику богаче, но не являются обязательными для MVP.

Основной сценарий
Пользователь вводит свой SteamID.
Сервис получает список друзей через Steam API.
Пользователь выбирает друга или вставляет второй SteamID.
Выполняется анализ обоих профилей.
Пользователь получает красивый отчет.

 
SteamMatch Score
Главный показатель сервиса.
Например:
SteamMatch Score
87%
Отличные игровые напарники.
Предлагаемая формула:
30% — общие игры.
25% — совпадение игровых жанров.
20% — сходство распределения игровых часов.
10% — достижения.
10% — любимые разработчики.
5% — текущая игровая активность.
Вес коэффициентов можно менять после тестирования.

Разделы отчета
1 Совпадение библиотек
Показывает:
количество игр у каждого;
количество общих игр;
процент совпадения библиотек.
Например:
У тебя — 327 игр
У друга — 412 игр
Общие игры — 186
Совместимость библиотек — 57%

1.1 Общие игры
Для каждой общей игры отображается:
количество часов пользователя;
количество часов друга.
Это позволяет понять, действительно ли пользователи играют в одинаковые проекты.
Пример:
Terraria
Ты — 230 часов
Друг — 240 часов
Factorio
Ты — 850 часов
Друг — 15 часов

2. Любимые жанры
На основании Store API строится профиль интересов пользователя.
Например:
FPS
RPG
Strategy
После этого рассчитывается совпадение жанров.
Например:
Совпадение жанров — 76%

3. Игровой профиль
На основании количества игр и часов определяется стиль игры пользователя.
Добавить исключения, к примеру если человек наиграл 2000 часов в доту и кс, а в другие максимум 20-30, то не учитывать слишком выделяющиеся игры (если часов в игре больше в 5 раз чем в игре которая идет перед ней по часам и при этом игра одна такая (как-то по процентам посчитать, допустим занимает 5% библиотеки), то не брать ее в расчет, но при этом дать пользователю возможность также посчитать avg и с учетом исключений)
Например:
Общее количество часов.
Среднее количество часов на игру.
Количество игр с большим временем прохождения.
Можно сделать вывод:
предпочитает играть долго в несколько игр;
любит постоянно пробовать новые проекты;
играет редко;
играет практически ежедневно.

4. Глубина прохождения
Сервис показывает:
Количество игр:
более 1 часа;
более 10 часов;
более 50 часов;
более 100 часов.
Это характеризует стиль игрока.

5. Backlog
Добавить в шутку если у пользователя много игр в которые он вошел и вышел (определять по часам к примеру <1 и при этом прохождения игры занимает больше времени) ты мог бы купить столько-то пицц, насколько мог бы купить подписки нитро в дс,
Количество игр:
не запускал;
запускал один раз;
активно играл.
Можно определить:
Backlog King.

6. Achievement Hunter
Если профиль открыт, отображаются:
средний процент достижений;
процент достижений в общих играх;
количество полностью завершенных игр.

7. Gaming DNA
Сервис определяет игровой архетип пользователя.
Возможные типы:
Completionist
Collector
Explorer
Specialist
Casual
Hardcore
Indie Fan
Strategy Brain
RPG Lover
Multiplayer Fan
У каждого пользователя может быть несколько архетипов.

8. Совпадение интересов
Сравнение:
жанров;
разработчиков;
издателей;
Можно показать:
Совпадение любимых разработчиков — 82%

9. Рекомендации для совместной игры
Сервис показывает:
Игры, которые есть у обоих, но почти не запускались.
Игры, которые нравятся одному пользователю и которые есть у второго и наоборот
Игры, которые стоит пройти вместе.

10. Активность
Используется поле playtime_2weeks.
Можно показать:
За последние две недели:
Ты — 38 часов
Друг — 41 час
Если пользователь давно не играл, это также отображается.

11. Steam Replay
Итоговая статистика:
самая любимая игра;
самая старая игра;
самая новая игра;
топ-10 игр по времени;
среднее количество часов на игру;
распределение часов между жанрами.
